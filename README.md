# 🛡️ FHE 기반 군용 자율주행차량 해킹 탐지 프레임워크

안녕! 이 프로젝트는 **"군사용 무인자율차량(UGV)이 적에게 해킹당하는 것을 어떻게 안전하고 빠르게 막을 수 있을까?"**라는 질문에서 시작된 'AI 프로그래밍' 과목 기말 프로젝트야. 

배경 지식이 없어도 쉽게 이해할 수 있도록 프로젝트의 핵심을 딱 3가지로 정리해 줄게!

---

## 1. 우리가 풀려고 한 문제 (Problem)
최신 군용 자율주행차량은 GPS나 핸들을 조작하려는 적의 **사이버 공격(해킹)** 위험에 노출되어 있어.
이 해킹을 막기 위해 똑똑한 **인공지능(AI) 서버**에게 차량의 주행 데이터(속도, 위치 등)를 보내서 "지금 이거 해킹당한 상태야?"라고 물어봐야 해.

🚨 **문제점**: 하지만 군사 데이터는 1급 기밀이잖아? AI 서버로 데이터를 보낼 때 평문(그대로 읽을 수 있는 데이터)으로 보내면, 중간에 적이 가로채거나 AI 서버 자체가 털렸을 때 차량의 작전 위치가 몽땅 뽀록나게 돼.

## 2. 우리의 해결책: 마법의 상자 '동형암호(FHE)'
이 딜레마를 해결하기 위해 **완전동형암호(FHE, Fully Homomorphic Encryption)**라는 최신 암호 기술을 도입했어.

* **일반 암호화**: 금고에 문서를 넣고 잠가서 서버로 보냄. 서버가 문서를 분석하려면 **반드시 금고를 열어야(복호화) 함**. (이때 정보 유출 위험 발생!)
* **동형암호(FHE)**: 특수 장갑이 달린 투명 금고. 서버는 **금고를 열지 않고도(암호화된 상태 그대로)** 데이터를 만지고 AI 계산을 할 수 있음!
* **결과**: 군용 차량의 기밀 데이터는 아무도 못 보게 완벽히 보호되면서, 해킹 여부만 정확히 알아낼 수 있어.

## 3. 이 프로젝트가 대박인 이유 (핵심 성과)
사실 동형암호(FHE)는 계산 과정이 너무 복잡해서 **속도가 엄청나게 느리다**는 치명적인 단점이 있어. 자율주행차는 0.1초 만에 사고가 나는데, 암호 계산하느라 10초가 걸리면 아무 쓸모가 없겠지?

우리는 이 문제를 해결하기 위해 무겁고 느린 딥러닝을 버리고, 동형암호에 최적화된 **'초경량화 AI 모델(Logistic Regression)'**을 직접 설계하고 훈련시켰어.

### 📊 실험 결과 (진짜 해냈음!)
1. **정확도(Accuracy) 96% 유지**: 데이터를 아무리 복잡하게 암호화해서 뭉개버려도, 우리 AI는 해커의 공격을 **96%의 정확도로 완벽하게 잡아냈어.** (암호화를 안 했을 때랑 성능이 100% 똑같아!)
2. **실시간 방어 성공 (0.034초)**: 암호 연산 때문에 속도가 4,000배 느려지긴 했지만, 애초에 AI를 너무 가볍게 만들었기 때문에 전체 계산이 **단 34ms(0.034초)** 만에 끝났어. 자동차가 핸들을 꺾기(보통 100ms) 전에 해킹을 먼저 차단할 수 있다는 걸 증명한 거지!
